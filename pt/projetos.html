E-commerce Microservices & Event-Driven Architecture

Sistema de e-commerce distribuído construído com Go, React, RabbitMQ e Webhooks.

Este projeto demonstra um ecossistema completo de e-commerce baseado em uma moderna arquitetura de microserviços. A plataforma foi projetada para demonstrar como sistemas distribuídos podem se comunicar utilizando arquitetura orientada a eventos, mensagens assíncronas e webhooks.

O back-end é implementado em Go, seguindo os princípios de Clean Architecture, enquanto as aplicações de front-end são construídas com React e TailwindCSS. Os serviços se comunicam através do RabbitMQ, permitindo um processamento resiliente e desacoplado de pedidos e pagamentos.

O sistema inclui múltiplos serviços independentes, como uma API de E-commerce, uma API Administrativa responsável por orquestrar os fluxos de pagamento, e um Gateway de Pagamento simulado que processa transações e envia atualizações de status através de webhooks.

O projeto também inclui múltiplas interfaces de front-end: uma Storefront para clientes, um Painel Administrativo para gerenciamento de pedidos e pagamentos, e um Painel do Gateway para simular aprovações e rejeições de pagamentos.

Tecnologias: Go, React, RabbitMQ, MySQL, Docker, Webhooks, APIs REST, Arquitetura Orientada a Eventos, Microserviços, Clean Architecture.
